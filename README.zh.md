# 双引擎质量网关系统

一个综合质量网关系统，包含两个独立模块：

1. **事件接收器 (Event Receiver)** - 部署在外网 (10.4.111.141)，接收 GitHub Webhook 事件
2. **事件处理器 (Event Processor)** - 部署在内网 (10.4.174.125)，处理质量检查任务，集成 Azure DevOps 和 AI 智能分析

## 事件接收器

GitHub Webhook 质量检查服务，用于监控和处理 GitHub Pull Request 和 Push 事件，自动执行质量检查流程。

### 功能特性

- **GitHub Webhook 集成**：接收和处理 GitHub PR 和 Push 事件
- **智能事件过滤**：只处理合并到 main 分支的 PR 和 main 分支的 Push
- **多阶段质量检查**：支持基础 CI、部署、专项测试三个阶段
- **MySQL 持久化**：存储事件和质量检查数据
- **RESTful API**：完整的 API 接口用于查询和管理数据
- **Docker 支持**：容器化部署

### 快速开始

```bash
cd src/modules/event-receiver
./install_quality.sh
```

### 容器架构

| 容器 | 端口 | 说明 |
|------|------|------|
| **quality-server** | 5001 | GitHub webhook 质量引擎 |
| **quality-frontend** | 8081 | Web UI |
| **quality-mysql** | 3306 | MySQL 数据库 |

### 质量检查阶段

1. **基础 CI** (stage_order=1)
   - 编译检查
   - 代码规范检查
   - 安全扫描
   - 单元测试

2. **部署** (stage_order=2)
   - 部署检查

3. **专项测试** (stage_order=3)
   - API 测试
   - 模块端到端测试
   - Agent 端到端测试
   - AI 端到端测试

### API 参考

#### 事件管理

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/events` | 获取事件列表 |
| `GET` | `/api/events/:id` | 获取事件详情 |
| `PUT` | `/api/events/:id/status` | 更新事件状态 |
| `DELETE` | `/api/events` | 删除所有事件 |

#### 质量检查

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/events/:eventID/quality-checks` | 获取质量检查列表 |
| `PUT` | `/api/quality-checks/:id` | 更新质量检查状态 |
| `PUT` | `/api/events/:eventID/quality-checks/batch` | 批量更新质量检查 |

详细 API 文档请参考 [Event Receiver README](src/modules/event-receiver/README.md)。

## 事件处理器

事件处理器模块，负责处理来自 Event Receiver 的质量检查任务，集成 Azure DevOps 和 AI 智能分析。

### 核心功能

#### 主要特性
- **事件获取**：每 30 秒从 Event Receiver 获取所有事件
- **AI 资源匹配**：自动将任务匹配到 Azure DevOps 资源
- **Azure DevOps 集成**：在 Azure DevOps 上执行 Pipeline
- **AI 日志分析**：并发分析构建日志，智能合并结果
- **跳过执行**：资源可标记为可跳过，绕过执行
- **请求池管理**：全局 AI 请求池防止过载

#### AI 请求池
- **全局并发控制**：管理所有事件的总 AI 请求数
- **可配置池大小**：默认 50，最大 200
- **单事件并发**：每个事件默认 20，最大 50 个并发日志文件
- **自动等待**：池耗尽时事件自动等待

#### 资源管理
- **允许跳过**：资源可标记为可跳过
- **自动跳过任务**：匹配到可跳过资源的任务自动标记为跳过
- **Azure 配置**：每个资源单独配置组织、项目、Pipeline ID

### 快速开始

```bash
cd src/modules/event-processor
./deploy-event-processor.sh
```

### 容器架构

| 容器 | 端口 | 说明 |
|------|------|------|
| **event-processor-server** | 5003 (外部) | 后端 API 服务器 (内部: 5002) |
| **event-processor-frontend** | 8082 | 前端 Web UI |
| **event-processor-mysql** | 3307 | MySQL 数据库 |

### 任务执行流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                     Event Processor                                  │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 1. Event Fetcher (每30秒)                                │      │
│  │    GET /api/events?status=pending                        │      │
│  └──────────────────────────────────────────────────────────┘      │
│                           │                                        │
│                           ▼                                        │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 2. AI 资源匹配                                            │      │
│  │    - 将任务匹配到 Azure 资源                              │      │
│  │    - 检查资源是否允许跳过                                 │      │
│  └──────────────────────────────────────────────────────────┘      │
│                           │                                        │
│                           ▼                                        │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 3. Task Executor - Azure DevOps 或跳过                    │      │
│  │    - 如果 allow_skip: 标记为跳过                          │      │
│  │    - 否则: 运行 Azure pipeline，获取日志                  │      │
│  └──────────────────────────────────────────────────────────┘      │
│                           │                                        │
│                           ▼                                        │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 4. AI 日志分析 (并发)                                     │      │
│  │    - 从全局池获取 tokens                                  │      │
│  │    - 使用 AI 分析每个日志文件                             │      │
│  │    - 智能合并结果                                         │      │
│  │    - 释放 tokens                                          │      │
│  └──────────────────────────────────────────────────────────┘      │
│                           │                                        │
│                           ▼                                        │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 5. Result Writer                                          │      │
│  │    PUT /api/events/:id/quality-checks/batch              │      │
│  └──────────────────────────────────────────────────────────┘      │
│                           │                                        │
│                           ▼                                        │
│  ┌──────────────────────────────────────────────────────────┐      │
│  │ 6. Next Task Creator                                      │      │
│  │    - 当前任务完成时创建下一个任务                          │      │
│  └──────────────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────────────┘
```

### 配置说明

管理控制台 (http://localhost:8082) 提供以下配置：

| 设置项 | 说明 | 范围/默认值 |
|--------|------|-------------|
| AI 服务器 IP | AI 模型服务器地址 | - |
| AI 模型 | 用于分析的模型名称 | - |
| AI Token | API 认证令牌 | - |
| Azure PAT | Azure DevOps 个人访问令牌 | - |
| AI 并发数 (单事件) | 每个事件最大并发日志文件数 | 1-50 (默认: 20) |
| AI 请求池大小 (全局) | 所有事件的总并发 AI 请求数 | 1-200 (默认: 50) |
| 日志保留天数 | 日志文件保留时间 | 1-30 (默认: 7) |

**重要提示**：AI 请求池大小必须大于 AI 并发数。建议不超过 AI 模型实际并发能力的 2/3。

### 可执行资源

资源定义了质量检查的执行位置和方式：

| 字段 | 说明 |
|------|------|
| 资源类型 | 检查类型 (basic_ci_all, deployment_deployment 等) |
| 允许跳过 | 如果为 true，任务标记为跳过而不执行 |
| 组织 | Azure DevOps 组织 |
| 项目 | Azure DevOps 项目 |
| Pipeline ID | Azure Pipeline ID |
| Pipeline 参数 | Pipeline 执行的 JSON 参数 |
| 仓库路径 | Git 仓库路径 |
| 创建者 ID | 创建资源的用户 |

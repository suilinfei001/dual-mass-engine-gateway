# Event Processor

事件处理器模块，负责处理来自 Event Receiver 的质量检查任务。

## 功能概述

Event Processor 是双引擎质量网关系统中的内网模块，主要功能：

1. **事件获取**：每 30 秒从 Event Receiver 获取所有事件
2. **任务创建**：根据事件的质量检查项创建执行任务
3. **任务调度**：按阶段顺序执行任务
4. **状态监控**：监控运行中的任务状态
5. **结果回写**：将检查结果更新到 Event Receiver

## 质量检查阶段

### 基础 CI (stage_order=1)
- 编译检查 (compilation)
- 代码规范检查 (code_lint)
- 安全扫描 (security_scan)
- 单元测试 (unit_test)

### 部署 (stage_order=2)
- 部署检查 (deployment)

### 专项测试 (stage_order=3)
- API 测试 (api_test)
- 模块端到端测试 (module_e2e)
- Agent 智能体端到端测试 (agent_e2e)
- AI 端到端测试 (ai_e2e)

```
│  ├─ 如果 status == "fail":                                                       │
│  │   └─ scheduler.FailTask(task, reason)                                        │
│  │       ├─ 更新数据库 (status: failed)                                          │
│  │       └─ 更新 Event 状态为 failed                                             │
│  │                                                                              │
│  └─ 如果超时:                                                                     │
│      └─ scheduler.TimeoutTask(task, reason)                                     │
└─────────────────────────────────────────────────────────────────────────────────┘
```

#### 5. 任务取消流程

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  5. 任务取消流程                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  触发场景:                                                                        │
│  ├─ PR 有新提交，需要取消旧任务                                                   │
│  └─ 手动取消任务                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
         │
         │
         ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  PR Handler - 取消旧任务                                                         │
│  ├─ 获取所有运行中的任务                                                          │
│  ├─ 对每个任务调用 CancelTask()                                                  │
│  └─ 调用 Mock Server 的取消接口                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
         │
         │ HTTP POST {task.CancelledURL}?task_id={task_id}
         │ 例如: POST http://localhost:8090/mock/basic-ci/cancel?task_id=xxx
         ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  Mock Server - 处理取消请求                                                       │
│  ├─ 停止定时器                                                                    │
│  ├─ 更新任务状态为 "cancelled"                                                    │
│  └─ 返回成功响应                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
         │
         │
         ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  Event Processor - 更新状态                                                      │
│  ├─ 更新数据库 (status: cancelled)                                               │
│  └─ 更新 Event Receiver 的 Quality Checks (status: cancelled)                    │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 关键要点总结

#### 1. Request URL 工作流程

```
创建任务时 → TaskCreator 生成 Request URL → 保存到 task.RequestURL
执行任务时 → Monitor 发送 GET 请求到 Request URL → Mock Server 创建任务并返回 task_id
监控任务时 → Monitor 定期查询 /mock/status/{task_id} → Mock Server 返回任务状态和结果
```

#### 2. Cancel URL 工作流程

```
需要取消时 → PR Handler 或其他触发器 → 发送 POST 请求到 Cancel URL
Mock Server → 停止定时器 → 标记任务为 cancelled
Event Processor → 更新数据库和 Event Receiver 状态
```

#### 3. Mock Server 模拟机制

- **接收请求**: 立即返回 task_id，任务状态为 "running"
- **模拟执行**: 使用定时器模拟任务执行时间
  - basic_ci_all: 3秒
  - deployment_deployment: 1分钟
  - 其他测试: 2秒
- **返回结果**: 定时器触发后，生成模拟结果并更新状态

#### 4. 数据流向

```
Event Receiver (存储事件和质量检查)
    ↓
Event Processor (处理任务)
    ↓
Mock Server (模拟执行)
    ↓
Event Processor (更新结果)
    ↓
Event Receiver (更新质量检查状态)
```

## Docker 部署

Event Processor 支持使用本地 Docker 镜像部署，不依赖 docker-compose。

### 快速开始

```bash
cd src/modules/event-processor
./deploy-event-processor.sh
```

### 部署步骤

部署脚本会自动执行以下操作：
1. 构建 Docker 镜像（后端 + 前端）
2. 创建 Docker 网络
3. 启动容器

### 部署模式

| 模式 | 命令 | 说明 |
|--------|-------|------|
| 升级模式 | `./deploy-event-processor.sh` 或 `-u` | 更新容器，保留数据（默认） |
| 恢复模式 | `./deploy-event-processor.sh -r` | 完全重装，删除旧容器和数据 |

### 容器信息

| 容器 | 镜像 | 端口 | 说明 |
|--------|-------|------|-------|
| event-processor-server | event-processor-backend:latest | 5002 | 后端 API 服务 |
| event-processor-frontend | event-processor-frontend:latest | 8082 | 前端 Web 服务 |

### 环境变量

| 变量 | 默认值 | 说明 |
|--------|---------|------|
| `EVENT_RECEIVER_API` | `http://10.4.111.141:5001` | Event Receiver API 地址 |

### Docker 命令

```bash
# 构建镜像
cd src/modules/event-processor
docker build -f Dockerfile_server -t event-processor-backend:latest .
```
